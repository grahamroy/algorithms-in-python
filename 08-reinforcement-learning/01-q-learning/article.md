# Q-Learning — Learning to Act Without a Map

### *Algorithms in Python --- Reinforcement Learning, Part 1*

---

Every algorithm in this series so far has learned from a
*fixed dataset*. Supervised models learned a mapping from
labelled examples. Unsupervised models found structure in a
pile of points. Even the recommender systems of the last
track learned from a log of interactions that had already
happened. The data sat still; the algorithm looked at it.

Reinforcement learning breaks that frame. There is no dataset.
There is an **agent** in an **environment**, and the agent
learns by *doing* — taking an action, seeing what happens,
collecting a reward, and adjusting. The data is generated by
the agent's own choices, which means the agent partly controls
what it gets to learn from. It is the formalism behind
game-playing AI (AlphaGo, Atari, StarCraft), robot control, and the
RLHF that aligns large language models.

**Q-Learning** (Watkins, 1989) is the foundational
reinforcement-learning algorithm and the cleanest place to
start. It learns the *value* of taking each action in each
state — a table of numbers `Q(state, action)` — purely from
experience, with no model of how the environment works. It
does not know the rules; it does not know where the rewards
are; it does not know what its actions will do. It just acts,
observes, and updates one number at a time. Given enough
exploration, those numbers converge to the optimal action
values, and acting greedily on them is the optimal policy.

The remarkable theoretical fact — Watkins & Dayan proved it in
1992 — is that Q-Learning converges to the optimal policy
*regardless of how the agent explores*, as long as it keeps
trying every action in every state. This article builds
Q-Learning from first principles. We will set up the
Markov Decision Process, derive the Bellman optimality equation
the algorithm is solving, walk through the temporal-difference
update at its core, implement tabular Q-Learning from scratch
on a grid-world with a cliff, and finish with the leap to deep
RL that the rest of this track builds on.

---

## The Markov Decision Process

Reinforcement learning problems are formalised as a **Markov
Decision Process** (MDP), defined by:

- **States** `s ∈ S` — the situations the agent can be in
  (a position on a grid, a board configuration, a robot's
  joint angles).
- **Actions** `a ∈ A` — the choices available (move up/down/
  left/right, place a stone, apply a torque).
- **Transition dynamics** `P(s' | s, a)` — the probability of
  landing in state `s'` after taking action `a` in state `s`.
- **Reward** `R(s, a, s')` — the scalar feedback for that
  transition.
- **Discount factor** `γ ∈ [0, 1)` — how much future rewards
  are worth relative to immediate ones.

The "Markov" part is the key assumption: the next state depends
only on the *current* state and action, not the full history.
Where you go next depends on where you are now, not how you got
here.

The agent's goal is to learn a **policy** `π(a | s)` — a way of
choosing actions — that maximises the expected **discounted
return**:

```
G_t = R_{t+1} + γ R_{t+2} + γ² R_{t+3} + ...
```

The discount `γ` makes the infinite sum finite and encodes a
preference for sooner rewards. `γ = 0.9` means a reward ten
steps away is worth `0.9¹⁰ ≈ 0.35` of an immediate one.

---

## Action values and the Bellman equation

The central object is the **action-value function** `Q^π(s, a)`:
the expected return from taking action `a` in state `s` and
following policy `π` thereafter.

```
Q^π(s, a) = E[ G_t | s_t = s, a_t = a ]
```

The **optimal** action-value function `Q*(s, a)` is the best
achievable value over all policies. If we knew `Q*`, the
optimal policy would be trivial: in each state, take the action
with the highest `Q*` value. The whole problem reduces to
*finding `Q*`*.

`Q*` satisfies the **Bellman optimality equation** — a
self-consistency condition that says the value of an action now
equals the immediate reward plus the discounted value of acting
optimally next:

```
Q*(s, a) = E[ R(s, a, s') + γ · max_{a'} Q*(s', a') ]
```

Read it as: "the value of `(s, a)` is what I get right now,
plus the discounted value of the best thing I can do from
wherever I land." This recursive relationship is the
foundation of every value-based RL method. The `max` over next
actions is what makes it *optimality* rather than just
consistency with some policy.

If we knew the transition dynamics `P` and rewards `R`, we
could solve this equation directly with dynamic programming
(value iteration). The whole point of Q-Learning is that we
*don't* know them — and learn `Q*` anyway, from experience.

---

## The temporal-difference update

Q-Learning estimates `Q*` by **sampling**. The agent takes
action `a` in state `s`, observes reward `r` and next state
`s'`, and nudges its estimate `Q(s, a)` toward the Bellman
target:

```
target = r + γ · max_{a'} Q(s', a')
Q(s, a) ← Q(s, a) + α · ( target − Q(s, a) )
```

Three pieces:

- The **TD target** `r + γ · max_{a'} Q(s', a')` is a one-step
  estimate of the true value — the reward we actually got plus
  our current best guess of what follows.
- The **TD error** `target − Q(s, a)` is the surprise: how
  wrong our current estimate was.
- The **learning rate** `α` controls how far we move toward the
  target. Each update corrects a fraction of the error.

This is bootstrapping — updating a guess using another guess
(`max Q(s', a')` is itself an estimate, not a known value). It
sounds circular, but it provably converges, because the reward
`r` injects real information at every step and the `max`
propagates it backward through the state space over many
episodes.

The defining feature is the word **off-policy**. The update
uses `max_{a'} Q(s', a')` — the value of the *best* next action
— regardless of what action the agent *actually* takes next.
The agent can explore randomly, even act badly, and still learn
the value of the *optimal* policy. This is what the next
article (SARSA) changes, and the contrast is illuminating.

---

## Exploration vs exploitation

There is one catch the Bellman equation hides. To learn
`Q*(s, a)` for an action, the agent has to *try* that action.
If it always picks the action it currently thinks is best
(**exploitation**), it will never discover that a different
action is better (**exploration**). An agent that exploits too
early gets stuck in a comfortable rut; one that explores
forever never cashes in what it learned.

The standard resolution is **ε-greedy**: with probability
`ε` take a random action, otherwise take the greedy
(highest-Q) one. Start with `ε` high (explore aggressively
when you know nothing) and decay it over training (exploit
more as your estimates sharpen). It is crude but effective, and
it satisfies Q-Learning's convergence requirement — every
action keeps getting tried.

```
choose_action(s):
    if random() < ε:  return random action      # explore
    else:             return argmax_a Q(s, a)    # exploit
```

---

## The full algorithm

```
Q-Learning(episodes, α, γ, ε):
    Initialise Q(s, a) = 0 for all s, a
    for each episode:
        s = start state
        while s is not terminal:
            a = ε-greedy(s)
            take a, observe reward r and next state s'
            Q(s, a) ← Q(s, a) + α·(r + γ·max_{a'} Q(s', a') − Q(s, a))
            s = s'
    return Q
```

That is the entire algorithm. No model, no dataset, no neural
network — just a table of numbers, repeatedly nudged toward the
Bellman target as the agent wanders the environment.

## A worked example: the cliff

The companion script implements the classic **Cliff Walking**
grid-world (Sutton & Barto). The agent starts bottom-left,
must reach bottom-right, and the entire bottom edge between
them is a cliff: stepping onto it gives reward −100 and sends
the agent back to the start. Every other step costs −1, so the
agent wants the *shortest* safe path. The optimal route hugs
the cliff edge — one row up, straight across, down at the end.

```
DEMO 1 --- Tabular Q-Learning on Cliff Walking
  Grid              : 4 x 12  (cliff along the bottom edge)
  Episodes          : 500
  alpha=0.5  gamma=0.95  epsilon: 1.0 -> 0.05 (decayed)

  Learned greedy path length     : 13 steps (optimal)
  Mean return, last 50 episodes  : -43.7
  Optimal path return            : -13
  States with a learned policy   : 37 of 48
```

```
DEMO 2 --- Q-Learning vs random policy
  Random policy   mean return : -2159.0   (falls off the cliff a lot)
  Q-Learning      mean return :   -43.7   (greedy path optimal; online cost from exploration)
```

```
DEMO 3 --- The learned greedy policy (arrows)
  S = start, G = goal, # = cliff
  > > v v v > v v v v v v
  v v v v v v v v v v v v
  > > > > > > > > > > > v
  S # # # # # # # # # # G
  (the greedy route runs along row 3 — one step above the cliff —
   then drops into the goal: the optimal but risky cliff-edge path)
```

Three observations.

**Q-Learning finds the optimal path — and it hugs the cliff.**
The greedy policy is a 13-step route with return −13, matching
the theoretical optimum. Crucially it runs along the row
*directly above the cliff* (the bottom path of arrows), because
that is genuinely the shortest safe route and Q-Learning's
off-policy `max` learns the value of acting optimally,
fearlessly. The agent learned the cliff's layout it was never
told about, purely from falling off during exploration.

**But the online return is −43.7, not −13.** This is the famous
Cliff Walking result. While the *learned greedy path* is
optimal, the agent's *actual* return during the last 50
training episodes is far worse, because it still explores with
`ε ≈ 0.05–0.1`, and a random step taken while walking the
cliff edge plunges it off for −100. Q-Learning learns the
risky-optimal policy and then *pays for its own exploration*
right next to the hazard. Hold this number — −43.7 — it is the
single most important reason the next article exists.

**The improvement over random is still enormous.** A random
walker averages roughly −2159 per episode, tumbling off the
cliff constantly. Even with the exploration penalty,
Q-Learning is ~50× better. And only 37 of 48 states get a
confident policy — states off the optimal path are rarely
visited under a decaying `ε`, so their `Q` values stay noisy;
the off-policy `max` is what still propagates value back to
them.

SARSA, the on-policy method in Part 2, learns a *safer* path
one row further from the edge — sacrificing two steps of
optimality to avoid the exploration-induced falls — and
records a better *online* return as a result. That trade-off,
visible as the gap between −43.7 here and SARSA's number next
time, is the whole reason the next article exists.

## Big-O and complexity

![[BIG-O TABLE IMAGE]]

Tabular Q-Learning is cheap per step and bounded by the size
of the table:

**Per step**: `O(|A|)` — choosing an action and computing the
`max` over next actions both scan the action set. With small
action spaces (4 directions, a handful of moves) this is
essentially constant.

**Per episode**: `O(steps · |A|)`.

**Memory**: `O(|S| · |A|)` for the Q-table — one number per
state-action pair. This is the binding constraint: tabular
Q-Learning is only feasible when the state space is small
enough to enumerate. A 4×12 grid has 48 states; chess has
~10⁴⁴; Atari from pixels has effectively infinite states. The
table does not fit, and that is precisely the wall that **Deep
Q-Networks** (Part 3) break through by *approximating* `Q`
with a neural network instead of storing it.

**Convergence**: guaranteed to `Q*` under mild conditions
(every state-action pair visited infinitely often, learning
rate decayed appropriately) — but "guaranteed eventually" can
be many episodes for large state spaces.

## Real-world ML and AI connections

**The foundation of value-based RL.** Q-Learning is the root of
a whole family: Deep Q-Networks, Double DQN, Dueling DQN,
Rainbow — every one is Q-Learning with a function approximator
and stability tricks. Understanding the tabular version is
understanding all of them.

**Atari from pixels.** DeepMind's 2013/2015 DQN — the result
that launched the deep-RL era — is Q-Learning with a
convolutional network estimating `Q(s, a)` from raw frames,
plus experience replay and a target network. Same update rule,
neural `Q`.

**Game-playing AI.** The value-estimation half of AlphaGo and
its successors descends from this lineage. Estimating "how good
is this position" is the Bellman-value problem Q-Learning
introduced.

**Operations research and control.** Inventory management,
elevator scheduling, traffic-light control, dynamic pricing —
classic tabular and approximate Q-Learning applications where
the state space is small enough to handle.

**RLHF and language models.** The reinforcement-learning step
that aligns LLMs uses policy-gradient methods (Part 4 of this
track) rather than Q-Learning directly, but the value-function
machinery — estimating expected future reward — traces straight
back to the Bellman equation here.

**Robotics.** Tabular Q-Learning rarely survives contact with
continuous robot state spaces, but it is the conceptual
starting point taught before the function-approximation and
policy-gradient methods that robots actually use.

## When NOT to use tabular Q-Learning

**When the state space is large or continuous.** The Q-table
is `O(|S| · |A|)`. Chess, Go, Atari-from-pixels, any robot with
continuous sensors — the table does not fit. Use Deep
Q-Networks or policy-gradient methods that approximate the
value function.

**When you need a stochastic policy.** Q-Learning's greedy
policy is deterministic. Games with mixed-strategy equilibria
(poker bluffing, rock-paper-scissors) need a *stochastic*
policy — policy-gradient territory.

**When actions are continuous.** The `max_{a'} Q(s', a')`
requires enumerating actions. For continuous control (torques,
steering angles) you cannot take a max over infinitely many
actions cheaply; use DDPG, SAC, or other actor-critic methods.

**When exploration is expensive or dangerous.** Q-Learning
learns by trying — including bad actions. If falling off the
cliff is a real robot breaking, unconstrained exploration is
unacceptable; you need offline RL, safe exploration, or
simulation.

**When the environment is not Markov.** If the next state
depends on history the agent cannot observe (partial
observability), plain Q-Learning's assumptions break. Use
recurrent policies or belief-state methods.

## What comes next

Part 2 of the Reinforcement Learning track is **SARSA** — the
on-policy sibling of Q-Learning. The two algorithms differ by
a single term in the update: Q-Learning bootstraps from
`max_{a'} Q(s', a')` (the best next action, off-policy), while
SARSA bootstraps from `Q(s', a')` for the action it *actually*
takes (on-policy). That one change makes SARSA learn the value
of the policy it is *following* — including its exploration —
and on the cliff it produces a visibly *safer* path. It is the
cleanest illustration in all of RL of the on-policy /
off-policy distinction.

After SARSA comes Deep Q-Networks (Q-Learning with a neural
network), policy-gradient methods, and actor-critic.

wall: **Q-Learning learns the optimal action-value function
from experience alone, with no model of the environment, by
repeatedly nudging each Q-value toward the Bellman target;
its off-policy `max` lets it learn the optimal policy while
exploring however it likes — and the table it stores is
exactly the limit that deep RL exists to overcome**.
