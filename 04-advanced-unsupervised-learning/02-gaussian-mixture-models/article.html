K-Means (Part 1 of the basic Unsupervised track) made two quiet design choices. First, every point gets a hard assignment to exactly one cluster — no probabilities, no uncertainty, just cluster = 3. Second, the clusters are spherical because Lloyd’s algorithm minimises Euclidean distance to the nearest centroid, which is rotationally symmetric. Both choices are fine for the canonical 3-Gaussian-blob test case where they give the right answer. Both break on data that violates them.

Gaussian Mixture Models (GMMs; the theory goes back to Pearson, 1894; the modern EM-based fitting algorithm was formalised by Dempster, Laird & Rubin in 1977) drop both assumptions in one go. Each cluster is modelled as a multivariate Gaussian — a probability distribution with its own mean, its own full covariance matrix, and a mixing weight that controls how much of the overall data it accounts for. Clusters can be elliptical, oriented along arbitrary axes, and of different sizes. Every data point gets a soft assignment — a probability of belonging to each cluster rather than a single hard label.

The algorithm that fits a GMM is Expectation–Maximisation (EM) — one of the most important algorithms in classical machine learning, applicable to any model with hidden / latent variables. GMM is the canonical EM example, but the same machinery powers Hidden Markov Model training, missing-data imputation, document-topic models (LDA shares EM’s shape), and a wide range of other probabilistic methods. Understanding EM on GMMs unlocks the whole family.

This article builds GMMs from first principles. We will write down the generative model, introduce the latent assignment variables that make the likelihood hard, derive EM as the iterative algorithm that handles them, implement the whole thing from scratch in numpy, fit it to data where K-Means gets the wrong answer (anisotropic stretched Gaussians), compare with scikit-learn, and finish with model selection via BIC and the practical question of how many components to use.

The generative model

A Gaussian Mixture Model assumes the data was generated by:

  1. Pick a cluster k ∈ {1, ..., K} with probability π_k (the mixing weights, summing to 1).
  2. Sample a point from a multivariate Gaussian N(μ_k, Σ_k).

The marginal density at any point x is:

p(x) = Σ_{k=1}^K  π_k · N(x | μ_k, Σ_k)

where the multivariate Gaussian density is:

N(x | μ_k, Σ_k) = (2π)^(-d/2) · |Σ_k|^(-1/2)
                  · exp(-½ · (x - μ_k)ᵀ Σ_k⁻¹ (x - μ_k))

The parameters to learn:

  • π_k — K mixing weights, summing to 1.
  • μ_k — K · d mean coordinates.
  • Σ_k — K covariance matrices of size d × d each (so K · d² numbers; with the symmetry constraint it’s K · d · (d + 1) / 2).

Plug a fit GMM into the density formula and you get a smooth function over the feature space. This is the key conceptual difference from K-Means: a GMM is a probability density model, not just a partition. You can ask it “what is the probability of this query point under each cluster?” and get a real number back.

The challenge: how do you fit π, μ, Σ from data without knowing which cluster each point came from?

The latent-variable problem

Standard maximum likelihood would have us write down the log-likelihood and maximise it:

log L(θ) = Σ_i log( Σ_k π_k · N(x_i | μ_k, Σ_k) )

The sum inside the log makes this analytically intractable. You cannot push the log inside the sum, and taking gradients gives coupled non-linear equations with no closed-form solution.

The trick is to imagine that we knew which cluster each point came from. Introduce a latent variable z_i ∈ {1, ..., K} for each data point — the cluster assignment. If we knew z, the log-likelihood would factor and be easy:

log L(θ | z) = Σ_i [ log π_{z_i} + log N(x_i | μ_{z_i}, Σ_{z_i}) ]

Each cluster’s parameters could be estimated independently from the points assigned to it. The problem is we don’t know z. The EM algorithm trades the unknown z for its expected value given the current parameters, then re-estimates parameters under that expectation. Then it does it again. The iteration is guaranteed to increase the (true marginal) log-likelihood at every step.

The EM algorithm for GMMs

Initialise π, μ, Σ (typically via K-Means on the data). Then alternate two steps until the log-likelihood converges.

E-step (Expectation). Compute the responsibility γ_{ik} — the probability that point i came from cluster k, given the current parameters:

γ_{ik} = π_k · N(x_i | μ_k, Σ_k)
       / Σ_j π_j · N(x_i | μ_j, Σ_j)

This is a soft assignment — each point is fractionally assigned to every cluster, with the responsibilities summing to 1 for each point.

M-step (Maximisation). Re-estimate parameters using the responsibilities as soft counts:

N_k = Σ_i γ_{ik}                          # effective N points in cluster k
π_k = N_k / N
μ_k = (1 / N_k) · Σ_i γ_{ik} · x_i
Σ_k = (1 / N_k) · Σ_i γ_{ik} · (x_i - μ_k)(x_i - μ_k)ᵀ

These are exactly the weighted versions of the formulas you would use if you had hard assignments — every point contributes to every cluster, scaled by its responsibility.

Compute the log-likelihood with the new parameters; if it hasn’t improved by more than a small tolerance, stop.

Convergence. Dempster, Laird & Rubin proved EM monotonically increases the marginal log-likelihood at every step. Convergence to a local maximum is guaranteed; convergence to the global maximum is not (the problem is multimodal in θ-space, much like K-Means on its objective). The standard fix is to run multiple restarts with different initialisations and keep the best.

Covariance variants

The covariance matrix Σ_k is the most flexible part of the model — and the most expensive. Standard implementations let you constrain it:

  • full — each cluster has its own arbitrary d × d covariance matrix. Maximum flexibility; O(K · d²) parameters. The default for non-trivial problems.
  • tied — all clusters share a single covariance matrix but each has its own mean. Useful when clusters have similar shape but different locations.
  • diag — each cluster has an axis-aligned diagonal covariance (no off-diagonal correlations). O(K · d) parameters. Reasonable when features are roughly independent.
  • spherical — each cluster has a single scalar variance, yielding spherical Gaussians. O(K) parameters. This is essentially soft K-Means.

The bias-variance tradeoff is the standard one: full fits the data best but needs the most data per cluster to estimate robustly; spherical needs little data but cannot represent elongated clusters. The right choice depends on d, on sample size per cluster, and on whether you expect the clusters to have meaningfully different shapes.

A worked example

The companion script generates 3 anisotropic Gaussian clusters — stretched ellipses oriented at different angles, the kind of structure where K-Means’ isotropic assumption produces visibly wrong answers — fits both K-Means and a from-scratch GMM, and reports the metric of interest.

  Components    : 3
  Covariance    : full
  Init          : K-Means centroids

  Final log-likelihood : -1972.78
  ARI vs true labels   : 0.842

  Final log-likelihood : -1972.78
  ARI vs true labels   : 0.842
  Agreement (ARI) with from-scratch GMM : 1.000

DEMO 3 --- K-Means on the same data (for comparison)
  K-Means ARI vs true labels  : 0.334

DEMO 4 --- Model selection: BIC vs K
     K      BIC     log-likelihood
   ---    -----    --------------
     1   5175.3            -2571.7
     2   4668.8            -2299.2
     3   4054.3            -1972.8
     4   4078.8            -1965.8
     5   4110.1            -1962.3
     6   4130.6            -1953.3

Four observations.

GMM recovers the clusters with ARI 0.842. The three strongly anisotropic ellipses overlap meaningfully in the middle of the feature space, so even the “right” model misclassifies a small fraction of points near cluster boundaries. This is the realistic situation on real data — clean ARI = 1.000 only happens on artificially-separable toys. Multivariate Gaussians with full covariance are still the right model class; the residual confusion is genuine ambiguity, not modelling error.

The from-scratch and sklearn implementations land at the same local optimum. Identical log-likelihood (-1972.78) and ARI 1.000 between them. Sklearn converged in 9 iterations versus our 14 because of small differences in initialisation and stopping criteria, but the destination is the same.

K-Means gets ARI 0.334. On the strongly elongated ellipses, K-Means’ spherical assumption is decisively wrong: the perpendicular bisectors between centroids cut clean through individual clusters rather than between them. ARI 0.334 is barely better than random with three classes — this is the canonical “K-Means fails on anisotropic clusters” demonstration, and the failure is far more dramatic than on the basic-track moons.

BIC picks K = 3 — the right answer. The Bayesian Information Criterion is BIC = ln(N) · p - 2 · ln(L) where p is the number of parameters and L the maximum likelihood. Lower is better. The minimum at K = 3 (BIC = 4054) tells us GMM-with-EM has correctly identified the underlying model order without ever being told there are 3 clusters. For real data BIC is the standard tool for picking the number of mixture components.

Big-O and complexity

![[BIG-O TABLE IMAGE]]

EM iterations are dominated by the per-point Gaussian density evaluations:

E-step. For each of n points, evaluate each of K Gaussian densities — each evaluation is O(d²) for full covariance (matrix-vector products and a determinant), O(d) for diagonal covariance. Per-iteration cost: O(n · K · d²) for full, O(n · K · d) for diag / spherical.

M-step. Weighted sums over all points for each cluster’s mean and covariance. O(n · K · d²) for full.

Convergence. Typical EM runs converge in 10–50 iterations for well-separated clusters; degenerate cases (overlapping clusters, poor initialisation) can take hundreds.

Memory. O(n · K) for the responsibility matrix, O(K · d²) for the parameters. The responsibility matrix dominates on large datasets.

Practical limit. Full-covariance GMMs become numerically fragile and computationally expensive past d ≈ 50 or so — the covariance matrices have ~d² parameters each, and estimating them requires ~d² points per cluster minimum. For high-dimensional data, use diag covariance (which is linear in d) or run PCA first.

The model itself stores O(K · d²) parameters and supports fast prediction (O(K · d²) per new point for full covariance, the same as one EM iteration’s per-point cost).

Real-world ML and AI connections

GMMs appear constantly in classical ML and in modern probabilistic pipelines:

Speech recognition. GMM-HMM systems were the dominant speech-recognition architecture from the 1990s through the deep-learning revolution. Each HMM state’s observation distribution was a GMM over acoustic features. Deep neural networks have largely replaced this — but the conceptual shape (mixture of Gaussians as the per-state output model) persists in many speech and audio pipelines.

Background subtraction in computer vision. Stauffer & Grimson’s classic 1999 paper Adaptive Background Mixture Models for Real-Time Tracking models each pixel’s intensity as a mixture of Gaussians, identifies the background as the most stationary mixture components, and flags moving objects as outliers from the background model. Still widely used in surveillance and video analytics.

Anomaly detection. Fit a GMM to “normal” data; new points with low likelihood under the fitted model are anomalies. The full-density model gives you a principled, calibrated anomaly score — better than DBSCAN’s binary noise label when you need a continuous score.

Missing-data imputation. EM was originally formalised in Dempster, Laird & Rubin’s paper precisely for this — fit a joint distribution under the assumption that missing values are missing-at-random, and impute them from the conditional distribution given the observed values. GMM is one of the standard joint-distribution choices.

Density estimation as a goal in itself. GMMs are universal density approximators (sufficient K can approximate any density to arbitrary accuracy). When you need the density — for sampling, for likelihood computation, for hypothesis testing — and the dimension is modest, GMMs are a practical answer.

Speaker identification. The “Universal Background Model” approach in classical speaker verification fits a giant GMM on a large pool of speakers, then adapts it per-speaker for identification. Still used in some production systems.

Initialisation for harder methods. Many other algorithms (deep Bayesian methods, autoencoders, variational autoencoders, mixture-of-experts neural networks) initialise their latent structure via GMM-style clustering of the embedding space.

Foundation for variational inference. GMM-EM is the historical and conceptual precursor to modern variational inference and the VAE framework — same idea of optimising over a distribution of latent variables, scaled up to deep networks.

The pattern: GMMs are the right tool when you need probabilistic clustering, when clusters might be elliptical, or when you need a density over the feature space rather than just a partition.

When NOT to use GMMs

GMMs’ assumptions are real:

When clusters are non-elliptical. GMMs can only model clusters that look like multivariate Gaussians (ellipses in 2D). Curved manifolds, ring shapes, half-moons — these are DBSCAN / spectral clustering territory, not GMM.

When d is large relative to per-cluster sample size. Full-covariance GMMs need at least d² points per cluster to estimate covariances robustly. With d = 100 and N = 1000 points per cluster, you are on the edge. Use diag covariance, regularise with a prior, or reduce dimensionality first.

When the data is genuinely non-Gaussian within each cluster. Heavy-tailed distributions, multimodal “clusters” (actually mixtures themselves), bounded data — GMMs will either over-fit with many components or systematically misrepresent the data. Use kernel density estimation, non-parametric methods, or Dirichlet Process mixtures.

When you need a fast, simple clustering with no probabilistic interpretation. K-Means is faster, simpler, and just as good when its assumptions hold. Use GMM only when you specifically need the soft assignments, elliptical flexibility, or density-modelling capability.

When EM is getting stuck in bad local optima. GMM-EM is sensitive to initialisation; on hard problems it can return wildly different fits across restarts. Use K-Means++ init, multiple restarts (sklearn’s n_init), or move to a Bayesian GMM with better-behaved priors.

When you need rigorous Bayesian model selection. BIC is an approximation; full Bayesian model averaging or Dirichlet-Process mixtures (which let K be infinite and choose itself) are more principled when the model order is critical.
