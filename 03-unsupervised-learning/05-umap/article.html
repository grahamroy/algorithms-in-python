In Part 4 we built t-SNE — the non-linear visualisation algorithm that took over scientific posters and ML demos for nearly a decade. It works beautifully on a few thousand points and shows non-linear cluster structure that PCA cannot. But it has three well-known failure modes: it is slow beyond a few thousand points, it preserves local neighbourhoods at the cost of global structure (clusters end up in arbitrary positions relative to each other), and it has no transform method — new points cannot be embedded into an existing t-SNE plot without re-running the whole optimisation.

UMAP (Uniform Manifold Approximation and Projection; McInnes, Healy & Melville, 2018) is the algorithm that fixed all three. It is grounded in a different mathematical foundation — Riemannian geometry and fuzzy simplicial complexes — but the operational recipe is similar: build a graph in high dimensions, find a low-dimensional arrangement that matches it. UMAP’s loss is cross-entropy instead of KL divergence, its high-d graph includes adaptive local connectivity, and its optimisation uses negative sampling — and the combination is dramatically faster than t-SNE, preserves more of the global structure, and produces a re-usable transformation that can embed new points without re-fitting.

Since around 2020, UMAP has largely displaced t-SNE in single-cell RNA-seq (the dominant scientific application), in visualisation of word and sentence embeddings, and in any setting where the dataset is too large for Barnes–Hut t-SNE. It is also the default in Google’s TensorBoard projector, in many vector-database visualisation tools, and in the AnnData / Scanpy / Seurat single-cell stacks.

This article builds UMAP from first principles. We will explain the fuzzy-simplicial-complex view of the high-dimensional graph, walk through the cross-entropy optimisation in low dimensions, implement a minimal from-scratch version on a tiny dataset to make the algorithm tangible, compare with the official umap-learn library on the digits dataset (alongside t-SNE and PCA), and finish with the cases where UMAP is the right default — and the few where it is not.

The two-line summary

UMAP, like t-SNE, has a high-dimensional and a low-dimensional representation, and the algorithm minimises the gap between them. The difference is in the shape of those representations.

High dimensions. Build a weighted k-nearest-neighbour graph. For each point, the edge weight to its n_neighbors nearest neighbours is a smooth function of distance, calibrated so the local connectivity is uniform across the dataset (each point has approximately the same effective number of connected neighbours, regardless of local density). The resulting graph is a fuzzy simplicial complex — a weighted graph whose edge weights live in [0, 1] and represent the “probability” that two points are neighbours on the underlying manifold.

Low dimensions. Initialise points in 2-D (UMAP uses spectral initialisation by default — eigenvectors of the graph Laplacian). Compute analogous edge weights between the low-dimensional points using a parametric curve 1 / (1 + a · d^(2b)) — similar in spirit to t-SNE’s Student-t kernel, with a and b chosen to match a target min_dist parameter.

Loss. Cross-entropy between the high-d and low-d edge-weight matrices. Roughly: pairs that are strongly connected in high-d should also be strongly connected in low-d, and pairs that are weakly connected should stay weakly connected.

Optimisation. Stochastic gradient descent with negative sampling. For each iteration: pick a random edge in the high-d graph and pull its endpoints together in low-d (attractive force); pick a few random non-edges and push their endpoints apart (repulsive force). This is dramatically cheaper than t-SNE’s full-pairwise gradient.

The whole loop is O(n · m) per iteration with m the number of edges in the k-NN graph (m ≈ n · n_neighbors), so effectively linear in the dataset size. That is the speed win.

What UMAP changes compared to t-SNE

Three design choices distinguish UMAP from t-SNE in ways that matter operationally:

Adaptive local connectivity. t-SNE’s perplexity sets the effective neighbourhood size uniformly across the dataset. UMAP adapts: for each point it sets the local connectivity so the nearest neighbour is at distance zero (probability 1.0 of being a neighbour) and the n_neighbors-th neighbour is at the relevant distance for that point. This handles datasets with strong density variation much better — t-SNE tends to over-spread sparse regions, UMAP keeps them in proportion.

Cross-entropy instead of KL divergence. Both objectives penalise mismatch between high-d and low-d similarity, but they weight the costs differently. UMAP’s cross-entropy also penalises false connections — putting two points close in low-d that were not close in high-d. t-SNE’s KL only directly penalises missed connections. The result: UMAP holds onto inter-cluster gaps better.

Stochastic gradient descent with negative sampling. Each iteration only touches one positive edge and a few negative samples, instead of summing over all pairs. This is what makes UMAP fast — and what gives it a transform method: a fitted UMAP model can embed new points by running the same SGD procedure with the existing low-d coordinates frozen.

Global structure is better preserved. The combination of adaptive connectivity and cross-entropy loss means inter-cluster distances in a UMAP plot carry some signal about inter-cluster distances in the original space. Not exactly, and the algorithm authors are careful to caveat this, but substantially more than t-SNE.

The algorithm

A simplified UMAP fit loop, with the details of the fuzzy-simplicial-complex construction abstracted into helper functions:

fit(X, n_neighbors=15, min_dist=0.1, n_components=2,
    n_epochs=200):
    # 1. Build the high-d fuzzy simplicial complex
    knn = nearest_neighbors(X, k=n_neighbors)
    rho = distance_to_nearest_neighbor(knn)   # per-point
    sigma = adaptive_bandwidth(knn, rho,
                               target=log2(n_neighbors))
    high_weights = fuzzy_membership(knn, rho, sigma)
    # symmetrise (probabilistic OR): u + v - u*v
    P = high_weights + high_weights.T \
        - high_weights * high_weights.T

    # 2. Initialise low-d embedding spectrally
    Y = spectral_init(P, n_components)

    # 3. Curve parameters fit so 1/(1+a*d^(2b)) matches min_dist
    a, b = fit_curve(min_dist, spread=1.0)

    # 4. Optimise by SGD with negative sampling
    for epoch in 1..n_epochs:
        for edge in sample_edges(P):
            i, j = edge.endpoints
            # Attractive: pull i and j together
            grad = attractive_grad(Y[i], Y[j], a, b)
            Y[i] += lr * grad;  Y[j] -= lr * grad

            for _ in range(n_negative_samples):
                k = random_point()
                # Repulsive: push i and k apart
                grad = repulsive_grad(Y[i], Y[k], a, b)
                Y[i] += lr * grad

        lr *= 1 - epoch / n_epochs   # linear cooldown

    return Y

This is the conceptual algorithm. The official umap-learn implementation is JIT-compiled with Numba for speed, uses the PyNNDescent approximate-k-NN library for the initial graph (so the graph construction is itself sub-quadratic), and has many parameter knobs we will not cover. But the essential structure is what is above: high-d graph, low-d initialisation, SGD with negative sampling.

The parameters that matter

Two knobs do most of the work in practice:

n_neighbors (default 15). Analogous to t-SNE’s perplexity. Smaller values (5 to 15) focus on local structure and tighter clusters; larger values (50 to 200) preserve more global structure at the cost of cluster definition. The default 15 is a reasonable starting point for most problems and is much less sensitive than t-SNE’s perplexity — UMAP plots at n_neighbors = 10 and n_neighbors = 50 look much more similar than t-SNE plots at perplexity 10 vs 50.

min_dist (default 0.1). Controls how tightly packed the low-d clusters are. Small min_dist (0.0 to 0.1) gives tight, crisply-separated clusters good for visualising discrete classes. Large min_dist (0.5 to 1.0) spreads clusters out and preserves more of the continuous topology — useful for exploring smooth manifolds (e.g. images of faces under varying illumination, where there are no clean clusters but a continuous family).

Other parameters matter less:

  • metric — Euclidean by default. Cosine for normalised embeddings (Word2Vec, sentence transformers). Manhattan, Hamming, and many others supported.
  • n_components — 2 for plots, 3 for interactive 3D, but UMAP can also be used as a non-linear feature extractor with n_components = 10 or higher (this is where its transform method shines).
  • n_epochs — default is “auto” (200 for big data, 500 for small). Rarely needs tuning.

The combination n_neighbors=15, min_dist=0.1, metric='euclidean' is the sklearn-like sensible default and gives publishable plots on the vast majority of datasets.

Four observations.

On a small dataset like this one (n = 1797), t-SNE is actually faster than UMAP — 3.5 s vs 8.8 s. UMAP’s wins are asymptotic: graph construction has fixed overhead that pays off only at scale. On the 60k-point MNIST dataset, t-SNE takes minutes while UMAP runs in under a minute; on a million-point single-cell dataset, t-SNE is impractical while UMAP finishes in tens of minutes. For the small datasets that fit in a Jupyter cell, t-SNE is fine; for everything bigger, UMAP is the right tool.

UMAP’s KNN accuracy slightly beats t-SNE’s (97.6% vs 96.9%), which is typical — UMAP’s better global-structure preservation tends to produce slightly cleaner cluster boundaries.

UMAP can embed new points. Demo 4 fits UMAP on 80% of the data and projects the held-out 20% into the same 2-D space. KNN-in-2D on the held-out set is 98.1% — actually slightly better than the fit-on-everything run, because the smaller training set produced tighter, more separated clusters. t-SNE cannot do this at all. If your pipeline needs to visualise streaming data in a stable layout, UMAP is the only viable choice.

PCA is essentially instantaneous — milliseconds versus seconds for the non-linear methods. On data where the structure is linear it will be just as good. PCA remains the right first reach. UMAP is for when PCA is not enough and the dataset is too big for t-SNE.

Big-O and complexity

![[BIG-O TABLE IMAGE]]

UMAP’s effective scaling is dramatically better than t-SNE:

Graph construction. PyNNDescent’s approximate k-NN runs in roughly O(n^1.14) empirically — sub-quadratic in n. Exact k-NN would be O(n²) but is rarely needed.

Optimisation. Each epoch touches O(n · n_neighbors) edges, plus a fixed number of negative samples per edge. Total optimisation cost is O(n · n_neighbors · n_epochs) — linear in n for fixed n_neighbors.

Memory. O(n · n_neighbors) for the sparse graph. The quadratic full-pairwise representations that t-SNE needs are avoided entirely.

In practice UMAP handles 100,000 points in a minute or so and millions of points within the practical limits of single-node analysis. For comparison, sklearn t-SNE struggles past 50,000 points even with Barnes-Hut.

What UMAP preserves and does not

UMAP shares many of t-SNE’s caveats — visualisations of non-linear projections are inherently lossy — but the quantitative behaviour differs in important ways:

Better preserved. Global structure. The relative positions of well-separated clusters in a UMAP plot tend to carry signal about their distances in the original space, much more than t-SNE. Still not exact — do not measure distances directly — but useful.

Better preserved. Cluster sizes. UMAP’s adaptive connectivity means dense regions of the original data stay dense in the embedding, and sparse regions stay sparse.

Same problem. Local distances inside a cluster are not metric in the embedding. The reasons that distances in a 2-D projection of high-d data are inherently distorted apply equally to UMAP and t-SNE.

Same problem. Random initialisation produces somewhat different layouts run-to-run. UMAP is more stable than t-SNE (spectral init helps), but for production-grade reproducibility you still need to pin the random seed.

UMAP-specific. Mathematical foundations rely on manifold-uniform-density assumptions that real data does not satisfy. The McInnes et al paper is careful about this; the algorithm works better than the theory strictly licenses, an empirical mystery that has not been fully resolved.

The honest summary: UMAP is t-SNE with most of the caveats turned down. It is the right default for new projects; t-SNE remains in use mostly out of inertia and for consistency with older publications.

Real-world ML and AI connections

UMAP’s footprint has grown rapidly since 2018:

Single-cell RNA-seq. The dominant application. Scanpy (Python) and Seurat (R) — the two standard single-cell analysis frameworks — both default to UMAP for visualisation, and most single-cell papers since 2020 use UMAP rather than t-SNE.

LLM and embedding visualisation. Visualising clusters of sentence-BERT embeddings, OpenAI embedding vectors, or any modern semantic embedding space is almost always UMAP these days. The combination of speed (millions of vectors) and global structure preservation makes it the right tool.

Vector-database exploration. Pinecone, Weaviate, Chroma, and friends typically ship UMAP visualisations as part of their debugging UIs.

Deep representation analysis. Inspecting what a deep model has learned — passing inputs through the network, collecting penultimate-layer activations, projecting to 2D — used to be a t-SNE workflow, now overwhelmingly UMAP.

Anomaly inspection. Outliers in UMAP plots stand out naturally. Coupled with HDBSCAN on the original (or PCA-reduced) data, UMAP visualisation plus density-based clustering is the standard “look for unusual structure in this dataset” recipe.

As a feature extractor. With n_components = 20 or 50, UMAP can replace PCA as the dimensionality-reduction step in a larger ML pipeline. The fact that it preserves non-linear structure makes it stronger than PCA for downstream classification or clustering on data with curved manifolds. This use is growing.

Topological data analysis. UMAP’s foundations in algebraic topology mean it integrates cleanly with topological-data-analysis (TDA) workflows — persistent homology, Mapper, and similar techniques where the shape of the data matters.

The pattern: UMAP is now the default visualisation algorithm for high-dimensional data, and increasingly also a default non-linear dimensionality reducer in production pipelines.

When NOT to use UMAP

UMAP is a strong default, not a universal one:

When you need a linear projection. PCA is faster, gives an interpretable orthonormal basis, has well-understood properties, and is the right choice when the structure is linear. Use UMAP only when you suspect non-linearity matters.

When you specifically need t-SNE behaviour. For replication of published t-SNE plots, or in workflows that have been calibrated to t-SNE’s idiosyncrasies, switching tools is not free. t-SNE is still a fine algorithm — UMAP is just usually better.

When you need a true clustering. UMAP is a visualisation and dimensionality-reduction algorithm, not a clustering one. Run HDBSCAN, K-Means, or hierarchical clustering on the original features (or PCA-reduced versions); use UMAP only to see what those clusters look like.

When determinism matters absolutely. The SGD has random elements, and different seeds give different layouts. Pin seeds carefully and document them.

When the dataset is genuinely tiny. Below ~200 points the overhead of building the k-NN graph dominates and PCA is fine.

Production embedding pipelines that need fast inference. The transform method exists but is slower than PCA’s matrix multiply. For latency-sensitive inference, train a feed-forward network to mimic the UMAP embedding (parametric UMAP) or use PCA.
